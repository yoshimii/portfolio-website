Stretch Goal: Responsive Web Design II


I didn't find many JavaScript animations that would have been useful to tweak in my template. It seems most of them were done in CSS. I did, however, find a function I've come across before but didn't understand so well: setTimeout()


I knew this particular use of the function delays page load to prevent lag by allowing things like images to load before showing the page.

I saw evidence of this in the stylesheet and understood setTimeout() better by looking closer at the class selector body.is-preload.

I noticed that the values for all the animations listed inside the body.is-preload class selector were set to none with an applied !important property which overrides any other set rules to those animations.

I also noticed that removeClass is used to remove the .is-preload class from the body which removes that sweet fade animation entirely.

But why would we turn off all animations?

Well, by looking at our setTimeout function we aren't turning them off forever. We are only turning them off for 100 miliseconds or .1 second. During this time we're giving the onload event time to trigger. Soooo once everything is loaded (onload triggered) we are safe to load our animation and reveal our wonderful contents.

But why wait?
Well, how much less epic would landing on my page be if you didn't have an aerial view of Mt. Fuji there to greet you? A lot.
